Guide
What this page is for

The GlidePress plugin is installed on WordPress sites with Composer, from the private package repo on this same domain. A token is the password that lets one site (or person) access that repo. Create one token per site/client — then you can revoke a single site's access without touching the others.

Walkthrough: onboard a new site
  1. Create a token above with a label that names the site or client (e.g. client-acme). Two copy buttons appear: the raw token and a ready-made composer config command.
  2. Send both snippets below to whoever manages that site, through a private channel (password manager share — not plain email if avoidable).
  3. On the site, they add the repo and plugin to the composer.json in the WordPress root: 
    {
    "repositories": [
        { "type": "composer", "url": "https://" }
    ],
    "require": {
        "composer/installers": "^2.0",
        "glidepress/glidepress-slider": "^2.1"
    },
    "extra": {
        "installer-paths": {
            "wp-content/plugins/{$name}/": ["type:wordpress-plugin"]
        }
    }
}
  4. They store the credentials once (outside the project, never committed): 
    composer config --global http-basic. token <TOKEN>
  5. Install and activate: 
    composer install
wp plugin activate glidepress-slider   # or via wp-admin → Plugins
Updates & new releases

Releases are published automatically when a version tag is pushed to the plugin repo (npm run release -- patch there). Nothing to do on this page. Consumers pull the newest allowed version with:

composer update glidepress/glidepress-slider
Revoking access

Click Revoke next to a token. The site using it loses access within about a minute (edge cache propagation) — its next composer install/update fails with a 401. The plugin already installed on that site keeps working; revocation only stops future downloads. To restore access, create a fresh token and send it over.

Troubleshooting a consumer
  • 401 Could not authenticate — token missing, mistyped, or revoked; redo the composer config step.
  • Could not find package — the repositories entry is missing from their composer.json.
  • Plugin ends up in vendor/ — they're missing composer/installers or the installer-paths block.
